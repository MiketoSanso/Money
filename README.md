# Фотостудия «Объектив» — Система управления заказами и клиентской базой

## Описание проекта

Веб-приложение для автоматизации работы фотостудии.

## Технологии

- Python 3.10+
- Flask 2.3
- PostgreSQL + SQLAlchemy
- Bootstrap 5 + FullCalendar.js

## Установка и запуск

Открой командную строку и выполни:

    cd C:\Users\Maksim\OneDrive\Desktop\MoneyProjectr
    venv\Scripts\activate
    python app.py

Открой браузер: `http://localhost:5000`

# ИНСТРУКЦИЯ ПО ТЕСТИРОВАНИЮ

## 1. Вход в систему

**URL:** `http://localhost:5000/login`

1. Открыть страницу → форма входа
2. Ввести `admin` / `admin123` → переход на `/dashboard`
3. В шапке видно имя пользователя и роль

## 2. Панель управления (Dashboard)

**URL:** `http://localhost:5000/dashboard`

1. Посмотреть карточки → 4 карточки с цифрами
2. Нажать «+ Новый клиент» → переход на форму создания клиента
3. Нажать «+ Новый заказ» → переход на форму создания заказа
4. Нажать «Расписание» → переход в календарь

## 3. Клиенты (CRM)

**URL:** `http://localhost:5000/clients`

**Просмотр:**
1. Открыть страницу → таблица со всеми клиентами

**Добавление:**
1. Нажать «+ Добавить клиента» или перейти на `/clients/create`
2. Заполнить поля (ФИО, телефон, email, дата рождения)
3. Нажать «Сохранить» → клиент появился в списке
4. Попробовать добавить клиента с тем же телефоном → ошибка

## 4. Заказы

**URL:** `http://localhost:5000/orders`

**Просмотр:**
1. Открыть страницу → таблица со всеми заказами

**Создание заказа:**
1. Нажать «+ Новый заказ» или перейти на `/orders/create`
2. Выбрать клиента из списка
3. Выбрать фотографа
4. Добавить услугу (выбрать из списка, указать количество)
5. Нажать «Создать заказ» → редирект на список заказов

**Проверка авторасчёта:**
1. При создании заказа в консоли видно, что цена рассчиталась
2. В списке заказов отображается итоговая сумма

**Проверка скидки:**
1. У клиента есть поле скидка (discount_percent)
2. При создании заказа сумма автоматически уменьшается на % скидки

**Смена статуса:**
1. В списке заказов есть выпадающий список статусов
2. Выбрать другой статус → страница обновляется, статус изменился
3. Доступные статусы: Новый → Подтверждён → В работе → Готов → Выдан → Оплачен → Отменён

## 5. Календарь и бронирование

**URL:** `http://localhost:5000/calendar-full`

**Просмотр календаря:**
1. Открыть страницу → виден календарь на месяц
2. Занятые слоты отображаются красным

**Бронирование слота:**
1. Выбрать фотографа (по умолчанию Иванов Иван, ID=2)
2. Выбрать дату
3. Выбрать время (10:00-19:00)
4. Указать ID существующего заказа
5. Нажать «Забронировать» → сообщение «Слот забронирован!»

**Запрет двойного бронирования:**
1. Попробовать забронировать тот же слот повторно → ошибка «Слот уже занят!»

## 6. Отчёты и аналитика

**URL:** `http://localhost:5000/reports`

1. Выбрать период (дата начала и конца)
2. Нажать «Показать»
3. Проверить: общее количество заказов, выручка, средний чек
4. Посмотреть топ-5 клиентов
5. Посмотреть статистику по фотографам (количество заказов, выручка)

## 7. Email-уведомления

**Проверка в консоли:**
1. Создать новый заказ
2. Посмотреть в терминале (где запущен `python app.py`)
3. Должна появиться строка: `📧 Email to client@email.com: Заказ создан - Ваш заказ #X на сумму X руб. принят.`

## 8. Ролевая модель

**Проверка ограничений:**
1. Выйти из системы (кнопка «Выйти»)
2. Зайти как `photographer` / `photo123`
3. Попробовать зайти на `/users` → доступ запрещён (только для админа)
4. Остальные страницы доступны

## 9. Выход из системы

1. Нажать кнопку «Выйти» в шапке
2. Перенаправление на страницу входа `/login`
3. Без авторизации доступ к страницам закрыт
